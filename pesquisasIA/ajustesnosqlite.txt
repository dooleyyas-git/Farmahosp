Pergunta 3:
ok mas não entendi esse "AS IDENTITY"

Resposta 3:
"AS IDENTITY" significa que o banco **gera automaticamente o valor do id**, não precisando informar manualmente ao inserir registros.
- ALWAYS: você **não pode** informar manualmente
- BY DEFAULT: você **pode** informar se quiser
Exemplo prático de inserção sem informar id.

Pergunta 5:
ok faça isso tudo com os comandos genericos

Resposta 5:
Script completo genérico:

CREATE SCHEMA integrador;

CREATE TABLE integrador.funcionarios (
    id INTEGER PRIMARY KEY,
    cpf VARCHAR(20) UNIQUE NOT NULL,
    senha VARCHAR(255) NOT NULL
);

CREATE TABLE integrador.medicamentos (
    id INTEGER PRIMARY KEY,
    nome VARCHAR(255) NOT NULL,
    quantidade_atual INTEGER DEFAULT 0
);

CREATE TABLE integrador.movimentacoes (
    id INTEGER PRIMARY KEY,
    id_medicamento INTEGER NOT NULL,
    tipo VARCHAR(10) NOT NULL,
    data_hora TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    CONSTRAINT chk_tipo CHECK (tipo IN ('entrada', 'saida')),
    CONSTRAINT fk_medicamento FOREIGN KEY (id_medicamento) REFERENCES integrador.medicamentos(id)
);

Alterações:
- Removeu USE SCHEMA e GENERATED AS IDENTITY
- Usou integrador.nome_tabela para indicar schema explicitamente

Pergunta 6:
mas vai funcionar sem o USE?

Resposta 6:
Sim, porque você usou integrador.funcionarios explicitamente. 
- SQL usa schema padrão se não for informado
- Referenciar schema.tabela elimina necessidade de USE
- Recomenda-se sempre indicar schema para portabilidade

---

Pergunta 7:
mas como o sql vai saber onde criar a tabela se eu n defini pra ele qual schema tem que ser usado?

Resposta 7:
Se não especificar schema, o banco usa **schema padrão** (ex: public, dbo). 
No script:
- CREATE TABLE integrador.funcionarios → explicitamente cria no schema integrador
- Se não indicar, criaria no schema default

---

Pergunta 8:
entendi, então não tem nada equivalente ao USE no sql genérico?

Resposta 8:
Exatamente, não existe USE no SQL padrão.
- Alternativas: 
  1) usar schema.tabela sempre
  2) alguns bancos têm SET SCHEMA (não universal)
Conclusão: schema.tabela é a forma genérica correta
